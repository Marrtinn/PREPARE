事前準備
  • 個人簡介
    • 我今年28歲是一位上班族，在 2 年前第一次接觸到 Html、CSS 及 JavaScrip，那時候的我剛離職沒多久 對寫程式非常的感興趣，不過礙於經濟上的壓力及父母的不諒解，練習了 1 個月便又回到上班族的生活，剛進到公司 還是菜鳥的我也無暇繼續練習，直到最近工作較為穩定，想在在職的情況下拚盡全力學習，期望能轉換跑道。
  • 為什麼想成為工程師
    • 除了對寫程式非常嚮往外，坦白說薪水比較優渥也是主要原因之一，另一方面這是一個帶著走的技術，在現在資訊發達且爆炸的時代，學會不管前端、後端、全端工程師 都是個國際通用的語言，在觀察 ex,Dcard、Facebook、Foodpand、Uber...的介面後，對可以設計出這樣的版型及方便使用者體驗的直覺操作而感到興趣 期許自己可以在上完課後針對前端工程有更深入得了解，未來也希望往全端工程師目標努力。
  • 為了成為工程師，曾經做過什麼努力?
    • 在第一份工作離職後，曾短暫的自學Html、CSS、JavaScrip (約一個月的時間)，在一開始連這些名詞都不懂得情況下，到後來慢慢的釐清這些語法的作用， 尤其在網上看教學的影片，爾後自己實際練習的結果，印象十分的深刻，但這都只是皮毛中的皮毛，針對未來的課程我會抽出更多時間參與練習，彭老師說 一個禮拜40-60hr才能夠跟上本科系的腳步，雖然上班時間卡了大半，但我會拚盡全力。
  • 對於工程師的職業想像是什麼？
    • 絕對不是一份輕鬆的工作，而是一份一直追求新的挑戰的職業，我能想像的是...未來自己 Hold 一份 Project 或解決大大小小 Bug Issue 所帶來得成就感 是我目前工作無法給予我的，另外在 Project 推進的過程，團隊互相討論砥礪、互相成長的過程，是我十分嚮往的。
  • 如果參與這個計畫，會怎麼安排學習時間？
    • 只要下班便會抓緊時間練習，在不加班的情況下，每天 6:00pm 開始學習，到 12:00pm，隔天早上 6:30 - 7:30 思考昨天遇到的問題並整理，如果有加班的 情況下，會在周末花時間盡可能跟上進度。
  • 如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？

    • 1.專業上的不熟悉 A:未來遇到哪些專業不足，針對去了解學習。

    2.時間不足 A:給自己進度計畫，或老師給的功課解決後,知其然知其所以然,下次再遇到類似問題,可以更有效率解決。

    3.無法馬上解決當前遇到的課題 A:會在每周一開會時提問，或是上各大論壇提問
  • 想要對我們說的事情？
    • 有幸可以在 youtube 上看見彭老師的直播十分開心，因為已經找了很多相關的自學網站，絕大部分都是給予工具，或一些較分散的技術， 當下學完可以了解，但後續要做什麼?這能用來幹嘛?沒有一個明確的方向，除了基本功可以在此熟練之外，給予我正確的方向前進我覺得是最重要的。
